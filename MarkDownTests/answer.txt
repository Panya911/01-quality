<p>Процессор этого языка должен давать на выходе HTML-код страницы.
Ниже описаны примерные правила, как он это должен делать.</p>

<p>Последовательность из двух переводов строки, 
между которыми нет ничего кроме пробельных символов, начинает новый абзац. 
Каждый абзац должен быть заключен в теги &lt;p&gt;...&lt;/p&gt;</p>

<p>Текст <em>окруженный с двух сторон</em>  одинарными символами подчерка 
должен помещаться в тег &lt;em&gt;.</p>

<p>Любой символ можно экранировать, чтобы он не считался частью разметки. 
_Вот это_, не должно выделиться тегом &lt;em&gt;.</p>

<p><strong>Двумя символами</strong> — должен становиться жирным с помощью тега &lt;strong&gt;.</p>

<p>Внутри <em>выделения &lt;em&gt; может быть _</em>&lt;strong&gt;__ выделение_.</p>

<p>Текст окруженный <code>одинарными _обратными_ кавычками</code> (backticks) должен попадать в тег &lt;code&gt;,
и к нему не нужно применять остальные правила форматирования. 
То есть слово "обратными" не должно выделиться &lt;em&gt; внутри code.</p>

<p>Подчерки внутри текста c цифрами_12_3 не считаются выделением и должны оставаться символами подчерка.</p>

<p>__непарные _символы не считаются `выделением.</p>
